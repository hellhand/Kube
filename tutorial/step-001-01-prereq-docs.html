Task 001.01 – Document Prerequisites & SDK Verification

Write down what’s needed and how to verify it works.

What to capture
  • OS notes, Go version, Vulkan SDK/driver requirements.
  • Packages: X11/GLFW dev headers, glslc.
  • Verification commands and expected success (no errors).
  • Validation env hints: VK_LAYER_PATH, VK_ICD_FILENAMES, VK_INSTANCE_LAYERS.

Example doc snippet

Requirements:
- Go 1.24+, vulkaninfo works, glslc available.
- Debian/Ubuntu packages: libx11-dev libglfw3-dev mesa-vulkan-drivers vulkan-tools.

Verify:
  vulkaninfo | head   # should print GPU info, no errors
  glslc --version     # prints glslc version
  go version

Validation:
  # request validation (default). To disable:
  VK_VALIDATION=0 go run .

Walkthrough for beginners

Open docs/prerequisites.md (or a similar note) and paste a short checklist the whole team can run before coding. Spell out the intent of each command so readers know why they run it.

# docs/prerequisites.md (excerpt)
## Vulkan runtime
vulkaninfo | head -n 15   # confirms ICD and driver are installed

## GLSL compiler
glslc --version           # prints the bundled shaderc version

## Go toolchain
go version                # confirm 1.25.x in this repo

## Optional: validation toggle used by the app
# export VK_VALIDATION=0   # disable if your driver lacks layers

Why it matters: if any of these fail, the later Vulkan calls (instance creation, swapchain creation) will crash or report missing extensions. Capture the exact error messages in the doc so others can diagnose quickly.
